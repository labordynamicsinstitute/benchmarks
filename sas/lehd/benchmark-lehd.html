LEHD SAS benchmark

Origin of this code

This code was used by the Longitudinal Employer-Household Dynamics (LEHD) Program at the U.S. Census Bureau to test SAS performance on different hardware and OS platforms. Performance is primarily influenced by the storage hardware used, but the OS can influence it as well. It replicates typical SAS processing at LEHD. The code was developed by many people, including Lou Schwarz.

Previous results

Not all previous results are disclosable. This code has been run on anything that can run SAS: Tru64 on Compaq Alpha, Solaris 8 and 9 on Sun, Linux 2.6 kernel on SGI Altix, Dell 32bit hardware, and Intel 64bit hardware.

How to run this code

(This information is also stored within the SAS code) All data is generated internally. No external data is needed. Data will be written and read from the filesystem specified on the command line: 
	sas -work /myfastdisk test.sas
Keep separate logs for runs on different filesystems. As with all benchmarking, multiple runs are needed to obtain reliable answers. 5-10 is a good number of repetitions. Each repetition should have its own log.
